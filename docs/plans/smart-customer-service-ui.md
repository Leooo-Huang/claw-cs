## 信息架构

现有侧栏"客服工单"是单入口，迭代后展开为带子菜单的分组：

```
AI 工作台（侧栏）
├── Dashboard
├── 审批中心（badge: 待处理数）
├── 工作流画布
├── ─── 业务场景 ───
├── 市场洞察 (W1)
├── 营销内容工厂 (W2)
├── 智能客服 ▾                      ← 展开/折叠子菜单
```

**路由设计：**

## 用户流程

### 流程 1：首次配置渠道（W3d）

```
触发：用户首次进入智能客服任意页面，检测到无渠道连接
1. 用户看到全屏引导页："连接你的第一个客服渠道"
   - 6 个渠道图标卡片：淘宝、拼多多、Shopify、抖店、邮箱、模拟数据
2. 用户点击一个渠道类型（如"邮箱"）
3. 用户看到分步配置向导（Sheet 侧滑面板）：
   - 步骤 1/4：填写连接信息（IMAP 服务器、端口、账号、密码）
   - 步骤 2/4：测试连接 → 显示加载中 → 成功✓ / 失败✗+错误信息
   - 步骤 3/4：消息路由规则（可选，默认全部接收）
   - 步骤 4/4：发送测试消息 → 确认完成
4. 用户看到渠道卡片变为"已连接"状态，绿色状态灯
完成：用户可开始接收该渠道的客户消息
异常：测试连接失败 → 显示错误详情 + "检查配置"按钮，不阻止重试
```

### 流程 2：导入知识库（W3b）

```
触发：用户进入知识库页面，首次无知识规则
1. 用户看到空状态引导："导入你的第一批业务知识"
   - 三种导入方式卡片：上传文件 / 导入历史对话 / 直接对话教 AI
2a. 用户选择"上传文件"：
   - 拖拽区出现，支持 PDF/Excel/CSV/TXT
   - 用户拖入文件 → 上传进度条 → 解析中（骨架屏）
   - 解析完成：显示提取的候选规则列表（每条含条件+内容+标签）
   - 用户逐条或全选审核：✓入库 / ✗忽略 / ✎编辑后入库
2b. 用户选择"导入历史对话"：
   - 上传 JSON/CSV 对话记录
   - OpenClaw 分析中（进度提示："正在从 128 条对话中提取规则..."）
   - 提取完成：同上审核流程
2c. 用户选择"直接对话教 AI"：
   - 对话面板打开，用户输入自然语言（如"7天内可无理由退货"）
   - OpenClaw 回复规则预览卡片（条件+内容+标签）
   - 用户点击"确认入库" → 规则立即生效
完成：知识库有了初始规则，W3a 可开始使用
异常：文件解析失败 → 显示"第 X 行格式错误" + 成功部分仍可入库
```

### 流程 3：处理单条工单（W3a）

```
触发：渠道收到新消息 → 工单出现在列表中
1. 用户在左侧看到新工单卡片（蓝色未读点 + 渠道图标 + 客户名 + 消息摘要）
2. 用户点击工单 → 中间区域加载对话详情
3. 用户看到：
   - 客户消息气泡（顶部：意图标签"售后-退货" + 情感标签"不满"）
   - AI 回复草稿（蓝色背景，可直接编辑的文本区）
   - 草稿下方：引用的知识规则（折叠列表，点击展开查看规则内容）
   - 右侧面板：客户画像 + 关联订单信息
4a. 用户满意 AI 回复 → 点击"批准并发送" → 回复通过 P4 发送到原渠道
4b. 用户不满意 → 在草稿区直接编辑文字 → 点击"修改并发送"
   - 系统检测到语义修改 → 底部出现 Toast："检测到内容修改，学习为新规则？"
     - [学习] → 后台提取规则，出现在 W3b 待审区
     - [跳过] → 直接发送
   - 系统检测到仅措辞修改 → 不弹提示，直接发送
4c. 用户点击"拒绝" → AI 重新生成回复（使用不同策略）
完成：工单状态变为"已回复"，列表中灰化
异常：发送失败 → 工单标记"发送失败" + "重试发送"按钮
```

### 流程 4：对话增强知识库（W3b 对话增强 Tab）

```
触发：用户切换到"对话增强"Tab
1. 用户看到对话界面，顶部提示："用自然语言告诉 AI 新的业务规则"
2. 用户输入："如果客户说尺码不合适，建议他看尺码表，尺码表在商品详情页底部"
3. OpenClaw 回复规则预览卡片：
   ┌────────────────────────────┐
   │ 📋 候选规则                 │
   │ 条件：客户反馈尺码不合适    │
   │ 回复：建议查看尺码表，位于  │
   │      商品详情页底部...      │
   │ 标签：#售前 #尺码           │
   │ [确认入库]  [编辑后入库]    │
   └────────────────────────────┘
4. 用户点击"确认入库" → 规则立即出现在规则管理 Tab
5. 用户可继续输入更多规则...
完成：规则即时生效，下次相关工单会引用新规则
```

### 流程 5：批量审核工单（W3c）

```
触发：用户进入批量运营台
1. 用户看到紧凑列表：每行 = 复选框 + 渠道图标 + 客户名 + 消息摘要 + AI 回复预览 + 置信度条
2. 用户用筛选栏选择条件（如"置信度 > 90% 且意图=物流查询"）
3. 列表刷新为匹配项 → 用户点击"全选"
4. 用户点击"批量通过" → 弹出预检弹窗：
   ┌──────────────────────────────┐
   │ 🔍 批量预检 — 随机抽取 5 条   │
   │                              │
   │ 1. 张三：包裹到哪了？         │
   │    AI：您的包裹已到达...  ✓   │
   │ 2. 李四：快递几天能到？       │
   │    AI：预计3-5个工作日... ✓   │
   │ ...                          │
   │                              │
   │ [取消]        [确认全部通过]  │
   └──────────────────────────────┘
5. 用户确认 → 全部通过并发送 → 底部显示结果统计 Toast
完成：批量处理完成，显示"已处理 47 条，节省约 23 分钟"
异常：部分发送失败 → 显示"成功 45 / 失败 2" + 查看失败详情
```

### 流程 6：审核学习候选规则（W3b 待审区 Tab）

```
触发：W3a 中的修改产生了学习候选
1. 用户切换到"待审区"Tab → 看到候选规则列表
2. 每条候选显示：
   - 来源工单链接（可跳转查看上下文）
   - 原始 AI 回复 vs 人类修改后回复（diff 高亮）
   - 提取的候选规则（条件+内容+标签+置信度）
3. 用户操作：
   - [入库] → 候选变为正式规则，触发知识同步
   - [编辑后入库] → 打开编辑器修改规则内容后入库
   - [忽略] → 标记为已忽略，不影响知识库
   - 全选 + 批量操作
完成：候选规则处理完毕，待审数降为 0
```

## 页面详细设计

### W3a：工单中心

**用途：** 客服的核心工作界面——接收消息、审核 AI 回复、发送回复
**入口：** 侧栏"智能客服 > 工单中心"（默认入口）
**出口：** 引用规则 → 跳转 W3b | 渠道异常提示 → 跳转 W3d

**布局：三栏**

```
┌──────┬────────────────────────────┬──────────┐
│ 工单  │     对话区                  │ 上下文    │
│ 列表  │                            │ 面板     │
│ w-80  │     flex-1                 │ w-72     │
│      │                            │          │
│      │                            │          │
└──────┴────────────────────────────┴──────────┘
```

**左侧：工单列表（w-80）**

- 顶部：筛选 Tabs（全部 / 待回复 / 已回复） + 渠道下拉筛选器（全部/淘宝/拼多多/邮箱...）
- 搜索框（搜客户名/消息内容）
- 工单卡片列表（按时间倒序）：
  - 每个卡片：渠道小图标 + 客户名 + 消息摘要（单行截断）+ 时间 + 状态色点
  - 未读工单：左侧蓝色竖线
  - 当前选中：浅蓝背景
  - 待回复数量 badge 在 Tab 上

**中间：对话区（flex-1）**

- 顶栏：客户名 + 意图标签（Badge: "售后-退货"）+ 情感标签（Badge: "不满"）+ 渠道来源图标
- 客户消息区：
  - 客户消息气泡（左对齐，灰色背景）
  - 消息时间戳
- AI 草稿区（蓝色浅背景卡片，边框左侧橙色竖线表示待审）：
  - 可编辑文本区（`textarea` 或 `contenteditable`，默认显示 AI 生成的回复）
  - 折叠面板："知识引用"——列出 AI 引用的知识规则（每条显示条件+内容摘要，点击展开完整内容，点击规则 ID 跳转 W3b）
  - 折叠面板："推理过程"——AI 为什么选了这些规则、为什么这样回复
- 底部操作栏（固定在底部）：
  - `[批准并发送]`（绿色主按钮）
  - `[修改并发送]`（橙色按钮，仅在内容被编辑后激活）
  - `[拒绝重新生成]`（灰色按钮）
  - 右侧：快捷话术下拉（常用回复模板一键插入）

**右侧：上下文面板（w-72）**

- 客户画像卡片：
  - 客户名、渠道、历史工单数、上次联系时间
- 关联订单卡片（如有 orderId）：
  - 订单号、商品名、金额、状态、物流信息
  - "查看详情"折叠展开
- 快捷操作：
  - "查询订单状态"按钮（调用 API 刷新）
  - "查看客户历史"按钮（展开历史工单列表）

**组件清单：**
| 组件 | 说明 | 交互 |
|------|------|------|
| TicketFilters | 状态 Tab + 渠道下拉 | Tab 切换刷新列表 |
| TicketSearch | 搜索框 | 输入 debounce 300ms 搜索 |
| TicketCard | 工单卡片 | 点击选中，加载对话区 |
| CustomerMessage | 客户消息气泡 | 只读 |
| AiDraftEditor | AI 回复编辑区 | 可编辑文本，修改后激活"修改并发送" |
| CitedRulesPanel | 知识引用折叠面板 | 默认折叠，点击展开 |
| ReasoningPanel | 推理过程折叠面板 | 默认折叠，点击展开 |
| ActionBar | 底部操作栏 | 三个按钮 + 快捷话术下拉 |
| LearningToast | 学习提示 Toast | "修改并发送"后弹出，3s 自动消失或用户操作 |
| CustomerProfile | 客户画像 | 只读 |
| OrderInfoCard | 订单信息 | 折叠/展开 |

### W3b：知识库工作台

**用途：** 知识的全生命周期管理——导入、浏览、编辑、对话增强、审核候选
**入口：** 侧栏"智能客服 > 知识库" | W3a 引用规则点击跳转
**出口：** 规则来源工单 → 跳转 W3a

**布局：单栏 + 三 Tab**

```
┌───────────────────────────────────────────────┐
│  知识库工作台                   [导入文件▴]     │
├───────────────────────────────────────────────┤
│  [规则管理]   [待审区(3)]   [对话增强]         │
├───────────────────────────────────────────────┤
│                                               │
│  Tab 内容区                                    │
│                                               │
│                                               │
└───────────────────────────────────────────────┘
  侧边悬浮：知识统计卡片（规则总数/活跃/待审/废弃）
```

**Tab 1：规则管理**

- 顶部筛选栏：来源下拉（文档/对话学习/手动/全部）+ 标签多选 + 状态下拉（活跃/待审/废弃）+ 置信度滑块 + 搜索
- 规则表格：
  - 列：条件（主列，宽）| 内容（摘要，hover 显示全文）| 来源图标 | 标签 Badges | 置信度条 | 引用次数 | 操作
  - 冲突规则：行背景橙色高亮，左侧 ⚠️ 图标，hover 显示冲突说明
  - 行操作：编辑（内联）| 废弃 | 查看来源
  - 内联编辑：点击条件或内容单元格 → 直接编辑 → Enter 保存 / Esc 取消
- 顶部右侧："导入文件"下拉按钮（上传文件 / 导入对话记录）
- 点击"导入文件"→ 弹出 Dialog：
  - 拖拽上传区 + 文件类型说明
  - 上传后显示解析进度 → 候选规则预览列表 → 全选/逐条入库

**Tab 2：待审区**

- Tab 标签上显示待审数量 Badge
- 候选规则列表（卡片形式）：
  - 每张卡片：
    - 顶部：来源工单链接 + 提取时间 + 置信度
    - 左半：原始 AI 回复（灰色背景）
    - 右半：修改后回复（绿色背景）
    - diff 高亮：修改的部分用下划线标记
    - 底部：提取的规则预览（条件+内容+标签）
    - 操作：`[入库]` `[编辑后入库]` `[忽略]`
- 顶部：全选 + 批量操作（批量入库/批量忽略）

**Tab 3：对话增强**

- 聊天界面风格：
  - 消息列表区（用户消息右对齐，OpenClaw 回复左对齐）
  - OpenClaw 的回复包含规则预览卡片（嵌入在消息气泡中）
  - 规则预览卡片：条件 + 内容 + 标签 + `[确认入库]` `[编辑后入库]` 按钮
  - 规则入库后，卡片变为绿色"已入库 ✓"状态
- 底部：输入框 + 发送按钮
- 顶部提示文案："用自然语言教 AI 新规则，如：'退货运费由买家承担'"
- 历史对话保留（可向上滚动查看之前的对话）

**侧边统计（固定在右下角浮动）：**
- 小卡片：规则总数 / 活跃 / 待审 / 本周新增 / 平均置信度

**组件清单：**
| 组件 | 说明 | 交互 |
|------|------|------|
| RuleFilters | 多维度筛选栏 | 实时过滤 |
| RuleTable | 规则表格 | 排序+内联编辑+冲突高亮 |
| ImportDialog | 文件导入弹窗 | 拖拽+解析进度+候选审核 |
| CandidateCard | 待审候选规则卡片 | diff展示+入库/忽略 |
| ChatPanel | 对话增强面板 | 聊天+规则预览+入库确认 |
| RulePreviewCard | 规则预览卡片（嵌入聊天） | 确认入库/编辑 |
| KnowledgeStats | 知识统计浮动卡片 | 只读 |

### W3c：批量运营台

**用途：** 高峰期快速批量处理工单和规则
**入口：** 侧栏"智能客服 > 批量运营"
**出口：** 查看工单详情 → 跳转 W3a | 查看规则 → 跳转 W3b

**布局：单栏 + 两 Tab**

```
┌───────────────────────────────────────────────┐
│  批量运营台                                     │
├───────────────────────────────────────────────┤
│  [批量工单审核(12)]   [批量规则管理(5)]         │
├───────────────────────────────────────────────┤
│  筛选栏：渠道 | 意图 | 置信度区间 | 时间范围    │
├───────────────────────────────────────────────┤
│  [☐ 全选] [批量通过] [批量拒绝]   已选: 0      │
├───────────────────────────────────────────────┤
│  ☐ │ 🛒 │ 张三 │ 包裹到哪了？ │ AI: 您的.. │ ███░ 92% │
│  ☐ │ 📧 │ 李四 │ 能退吗？    │ AI: 7天内.. │ ██░░ 78% │
│  ...                                          │
├───────────────────────────────────────────────┤
│  📊 待处理: 12  今日已处理: 35  节省: 18分钟   │
└───────────────────────────────────────────────┘
```

**Tab 1：批量工单审核**

- 筛选栏：渠道多选 + 意图下拉 + 置信度滑块范围（如 80%-100%）+ 时间范围
- 批量工具栏：全选复选框 + "批量通过" + "批量拒绝" + 已选计数
- 紧凑列表（虚拟滚动，支持大量数据）：
  - 每行：复选框 | 渠道图标 | 客户名 | 消息摘要（truncate 40 字）| AI 回复摘要（truncate 50 字）| 置信度进度条
  - 行点击 → 底部展开该条详情（不跳转页面）
  - 置信度颜色：>90% 绿 | 70-90% 黄 | <70% 红
- "批量通过"按钮点击 → 预检弹窗（SpotCheckDialog）：
  - 标题："批量预检 — 随机抽取 5 条"
  - 5 条工单详情卡片（客户消息+AI回复完整内容）
  - 底部：`[取消]` `[确认全部通过(47条)]`
- 底部统计栏：待处理 / 今日已处理 / 预计节省时间

**Tab 2：批量规则管理**

## 响应式策略

本系统面向电商运营/客服主管，**桌面优先**（1280px+ 屏幕）。

| 断点 | 布局变化 |
|------|---------|
| ≥1280px | 三栏完整展示（W3a） |
| 1024-1279px | W3a 右侧上下文面板折叠为图标触发的浮动面板 |
| 768-1023px | W3a 变为两栏（列表+对话），上下文移入对话区顶部折叠 |
| <768px | 单栏，列表和对话区切换显示（不推荐，但不阻止） |

侧栏在 <1024px 时折叠为图标栏（w-16），hover 展开。

### W3d 渠道配置中心

| 数据 | 来源 API | 说明 |
|------|---------|------|
| 渠道列表 | `GET /api/channels` | 含 status, messageCount |
| 添加渠道 | `POST /api/channels` | body: type, name, config |
| 测试连接 | `POST /api/channels/[id]/test` | 返回成功/失败 |
| 删除渠道 | `DELETE /api/channels/[id]` | 需确认 |
| 渠道健康 | `GET /api/channels/[id]/health` | 连接状态+错误信息 |
| 渠道统计 | `GET /api/channels/[id]/stats` | 今日/本周/本月消息数 |
| 手动拉取 | `POST /api/channels/[id]/poll` | 开发阶段手动触发 |

### 跨页面 SSE 事件

| 事件 | 触发场景 | 影响页面 |
|------|---------|---------|
| `ticket:created` | 新消息从渠道进入 | W3a 列表刷新, W3c 计数更新 |
| `ticket:draft_ready` | AI 草稿生成完成 | W3a 对话区刷新 |
| `ticket:replied` | 回复已发送 | W3a 状态更新 |
| `knowledge:synced` | 知识同步到 OpenClaw 完成 | W3b 同步状态提示 |
| `learning:new_candidate` | 新的学习候选产生 | W3b 待审区 badge 更新 |
